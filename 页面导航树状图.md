# Secalender 页面导航树状图

本文档详细列出了 Secalender 应用中所有页面的导航关系，以树状图形式展示每个页面的点击跳转路径。

---

## 📱 应用入口

```
RootView (根视图)
├── AuthenticationView (认证视图) [未登录时显示]
│   ├── SignInEmailView (邮箱登录)
│   │   └── EmailVerificationView (邮箱验证)
│   └── BasicInfoView (基本资料填写)
│
└── ContentView (主容器) [已登录时显示]

        ├── Tab 2: 智能規劃
        ├── Tab 3: 朋友＆社群
        └── Tab 4: 功能
```

## 📊 导航方式说明

### 1. NavigationLink (导航链接)
- 使用 `NavigationLink` 实现的页面跳转
- 支持返回按钮和导航栈管理
- 示例：`MemberView` → `EditProfileView`

### 2. Sheet (模态弹窗)
- 使用 `.sheet()` 修饰符实现的模态展示
- 从底部向上滑出
- 示例：`CalendarView` → `EventCreateView`

### 3. FullScreenCover (全屏覆盖)
- 使用 `.fullScreenCover()` 修饰符实现的全屏展示
- 覆盖整个屏幕
- 示例：`AIPlannerView` → `PlanDetailView`

### 4. NavigationStack/NavigationView (导航容器)
- 用于包装需要导航功能的视图
- 提供导航栏和返回功能
- 示例：`EventShareView` 内部使用 `NavigationView`

## 🎨 特殊交互说明

### 多选模式
- **触发方式**：在 `CalendarView` 中长按事件进入多选模式
- **功能**：可以选择多个事件进行批量操作
- **操作**：
  - 批量分享 → `BatchShareEventsView`
  - 批量编辑 → `MultiEventView`

### 中间按钮动态功能
- **Tab 1 (行事曆)**：创建行程 → `EventCreateView`
- **Tab 2 (智能規劃)**：AI 对话 → `AIConversationView`
- **Tab 3 (朋友＆社群)**：功能菜单 → `FriendsActionBottomSheet`
- **Tab 4 (功能)**：创建行程 → `EventCreateView`

### 手势导航
- **TravelTemplateView**：支持左右滑动切换子 Tab
- **CalendarView**：支持下拉刷新

## 📌 注意事项

1. **导航栈管理**：某些页面使用 `NavigationView` 包装，确保导航功能正常
2. **环境对象**：大部分页面需要 `FirebaseUserManager` 作为环境对象
3. **状态管理**：使用 `@State`、`@Binding`、`@StateObject` 等管理页面状态
4. **异步操作**：数据加载使用 `async/await` 模式
5. **Sheet 切换**：某些复杂流程使用统一的 sheet 状态管理，避免状态错乱
